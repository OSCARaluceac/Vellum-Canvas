# Gestión del Proyecto - Vellum Canvas

## Metodología: Híbrido Agile
Para el desarrollo de Vellum Canvas, se ha implementado un sistema híbrido que combina la estructura de **Scrum** con la fluidez de **Kanban**.

### 1. Planificación (Enfoque Scrum)
El proyecto se ha dividido en "Épicas" o módulos funcionales:
- **Infraestructura:** Setup y arquitectura inicial.
- **Módulo Narrativo:** Motor de escenas y diálogos.
- **Módulo Geográfico:** Mapas y sistema de coordenadas.
- **Módulo Económico:** Inventario, tiendas y base de datos de ítems.

### 2. Ejecución (Enfoque Kanban)
Se utiliza un tablero en **Trello** para gestionar el flujo diario de tareas:
- **Backlog:** Ideas y funcionalidades pendientes de priorizar.
- **To Do:** Tareas técnicas desglosadas listas para ser iniciadas.
- **In Progress:** Tareas actualmente en desarrollo (máximo 2 simultáneas).
- **Review:** Código pendiente de pruebas y depuración de errores.
- **Done:** Funcionalidades 100% operativas y testeadas.

## Herramientas de Organización
- **Trello:** Visualización del flujo de trabajo.
- **GitHub Issues:** Seguimiento de bugs detectados durante la fase de testing.
- **Documentación Continua:** Actualización síncrona de los archivos en `/docs` conforme se completa cada hito.
